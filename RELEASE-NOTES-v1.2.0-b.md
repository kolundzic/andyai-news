# AndyAI News — v1.2.0-b — Archive / History Mode

This overlay introduces the first safe archive/history bridge for the newsroom operating model.

## What this pack adds

- archive/history shared types
- history snapshot helpers
- `/admin/history` landing page
- `/api/admin/history` JSON endpoint
- initial data snapshot for history/archive mode
- safe bridge with `/admin/v110a` preserved as preview/regression lane

## Why this matters

`v1.2.0` moves AndyAI News from a live MVP admin panel toward a structured multilingual newsroom operating system with:

- shared edition/day spine
- archive/history memory
- locale-aware content
- deterministic newsletter compose
- voice/TTS-ready prep

This `v1.2.0-b` cut focuses specifically on **archive/history memory** as the next minimal implementation step.

## Notes
