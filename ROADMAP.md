# AndyAI News Roadmap

**Project:** `andyai-news`  
**Positioning:** Local-first AI content operating system  
**Owner:** Andrija Kolundzic (Andy)

━━━━━━━━━━━━━━━━━━━━

## Purpose

AndyAI News is not being developed as a simple blog or a generic CMS.

It is being developed as a **content operating system** for:

- daily AI news production
- structured editorial workflow
- content validation
- publish discipline
- future multilingual expansion
- future voice-over / media expansion

The roadmap exists to keep the product **focused, incremental, and release-friendly**.

━━━━━━━━━━━━━━━━━━━━

## Product philosophy

The goal is not random feature growth.

The goal is to build a system that becomes:

- reliable for daily use
- easy to operate
- hard to break
- simple to publish
- expandable into a larger media engine

Every version should improve one of these layers:

1. operator comfort
2. publishing discipline
3. content reuse
4. product scalability

## v1.1 — Operator Upgrade Pack

**Goal:** make the daily workflow smoother, safer, and faster.

### Scope

#### 1. Multi-day navigation
Add easier movement between multiple daily datasets/issues.

Why it matters:
- turns single-day editing into real daily operations
- prepares the base for archive/history mode
- reduces manual file switching

#### 2. Draft autosave
Add local autosave during editing sessions.

Why it matters:
- reduces risk of losing work
- improves operator confidence
- makes the admin panel feel production-ready

#### 3. Manifest helper / live day switch
Add a simpler way to control which day is currently live.

Why it matters:
- reduces manual manifest mistakes
- improves publish discipline
- speeds up daily publishing

#### 4. Newsletter export
Add export mode for bulletin / email-ready output.

Why it matters:
- enables content reuse across channels
- creates immediate product value
- opens path toward newsletter operations

━━━━━━━━━━━━━━━━━━━━

## v1.2 — Expansion Pack Candidates

These are strong next candidates, but should come after v1.1 is stable.

### 1. Archive / history mode
Basic public or admin-side access to previous published days.

### 2. Multilingual layer
Schema or UI support for multiple language versions.

### 3. Voice-over / TTS prep
Prepare story output in a format ready for voice production.

### 4. AI-assisted drafting
Optional helper layer for generating or improving story drafts.

━━━━━━━━━━━━━━━━━━━━

## v1.3+ — Productization Direction

After the operator workflow becomes solid, AndyAI News can evolve toward a broader platform.

### Direction A — Editorial system
Internal workflow tool for structured daily AI reporting.

### Direction B — Publishing engine
A reusable engine for newsletters, magazines, and bulletins.

### Direction C — Media factory platform
A larger system for multilingual text, audio, visuals, and distribution.

## Release discipline

AndyAI News should grow in **small, controlled, versioned steps**.

Recommended release method:

- one improvement pack
- one ZIP
- one clear version
- one clean push
- one visible release note

That keeps the product:

- understandable
- reversible
- demonstrable
- professional

━━━━━━━━━━━━━━━━━━━━

## Canon release sequence

- `v1.1.0` — Operator Upgrade Pack
