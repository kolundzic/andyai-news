# AndyAI News Roadmap

**Project:** `andyai-news`  
**Positioning:** Local-first AI content operating system  
**Owner:** Andrija Kolundzic (Andy)

━━━━━━━━━━━━━━━━━━━━

## Purpose

AndyAI News is not being developed as a simple blog or a generic CMS.

It is being developed as a **content operating system** for:

- daily AI news production
- structured editorial workflow
- content validation
- publish discipline
- future multilingual expansion
- future voice-over / media expansion

The roadmap exists to keep the product **focused, incremental, and release-friendly**.

━━━━━━━━━━━━━━━━━━━━

## Product philosophy

The goal is not random feature growth.

The goal is to build a system that becomes:

- reliable for daily use
- easy to operate
- hard to break
- simple to publish
- expandable into a larger media engine

Every version should improve one of these layers:

1. operator comfort
2. publishing discipline
3. content reuse
4. product scalability

## v1.2 — Integration / Expansion Pack

**Goal:** turn the preview upgrade line into more deeply integrated product behavior.

### Strong candidates

#### 1. Main admin integration
Move selected `v110a` upgrade features into the main `/admin` workflow.

#### 2. Archive / history mode
Add support for browsing previous published issues more directly.

#### 3. Multilingual layer
Prepare data and UI structure for multiple language versions.

#### 4. Newsletter data binding refinement
Improve export quality and connect it more tightly to real issue/story content.

#### 5. Voice-over / TTS prep
Prepare export and story structure for voice production workflows.

━━━━━━━━━━━━━━━━━━━━

## v1.3+ — Productization Direction

After the operator workflow becomes more integrated, AndyAI News can evolve toward a broader platform.

### Direction A — Editorial system
Internal workflow tool for structured daily AI reporting.

### Direction B — Publishing engine
A reusable engine for newsletters, magazines, and bulletins.

### Direction C — Media factory platform
A larger system for multilingual text, audio, visuals, and distribution.

## Release discipline

AndyAI News should continue to grow in **small, controlled, versioned steps**.

Recommended release method:

- one improvement pack
- one ZIP
- one clear version
- one clean push
- one visible release note

That keeps the product:

- understandable
- reversible
- demonstrable
- professional

━━━━━━━━━━━━━━━━━━━━
