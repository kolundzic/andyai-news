# AndyAI News v1.1.0 — Consolidated Release

**Release type:** Consolidated milestone  
**Project:** `andyai-news`  
**Owner:** Andrija Kolundzic (Andy)

━━━━━━━━━━━━━━━━━━━━

## Summary

v1.1.0 consolidates the full **Operator Upgrade Pack** introduced across the `v1.1.0-a` to `v1.1.0-e` release line.

This release does **not** replace the main production admin route.

Instead, it completes a **safe preview / admin-isolated upgrade layer** at:

`/admin/v110a`

That layer now demonstrates the next generation operator workflow for AndyAI News.

━━━━━━━━━━━━━━━━━━━━

## Included packs

### v1.1.0-a — skeleton / UI slots / feature placeholders
Introduced the safe preview route and feature-card foundation.

### v1.1.0-b — multi-day navigation
Added manifest-backed issue discovery and working-date selection.

### v1.1.0-c — draft autosave
Added local-first autosave preview with restore / clear controls and visible save-state feedback.

### v1.1.0-d — manifest helper / live day switch
Added guided live-day switch preview with change detection, switch-plan summary, and simulation flow.

### v1.1.0-e — newsletter export
Added preview newsletter export with markdown / html / text formats, filename helper, copy action, and download action.

━━━━━━━━━━━━━━━━━━━━

## What v1.1.0 means

AndyAI News now has a complete **Operator Upgrade Pack preview layer**.

That means the project now includes a safe working surface for validating the next admin-generation workflow before merging those behaviors into the main production admin route.

This is important because it allows the product to evolve without destabilizing the existing live publishing workflow.

━━━━━━━━━━━━━━━━━━━━

## Product impact

With v1.1.0, AndyAI News becomes more clearly positioned as:

- a local-first editorial operating system
- a structured daily publishing workflow
- a platform that can evolve into newsletter, multilingual, and media-oriented publishing

The project is no longer just a stable MVP.

It now has a verified next-step operator model.
